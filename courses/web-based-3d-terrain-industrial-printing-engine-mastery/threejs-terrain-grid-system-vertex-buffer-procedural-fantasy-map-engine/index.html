The key idea is a conceptual shift:

Terrain is not a mesh — it is a structured data system.

Each vertex becomes an independent computational unit inside a dynamic grid.

2. PlaneGeometry as Terrain Foundation

PlaneGeometry is used as the base structure because it provides:

  • A flat surface in 3D space
  • Adjustable segmentation (resolution control)
  • A structured grid of vertices

Core Concept:

  • More segments → more vertices → higher detail
  • Fewer segments → fewer vertices → better performance

This introduces the first major trade-off in procedural terrain design: quality vs performance.


3. Vertex Grid Architecture

Terrain is constructed as a 2D grid mapped into a 1D memory array.

Structure:

  • Rows → vertical division
  • Columns → horizontal division
  • Intersection points → vertices

Each vertex represents a unique spatial coordinate in 3D space.

This grid structure enables deterministic terrain manipulation and procedural control.


4. BufferGeometry Position System

All vertex data is stored in:

  • BufferGeometry.position
  • A Float32Array

Each vertex contains:

  • X → horizontal position
  • Y → height (elevation)
  • Z → depth position

Key Transformation:

The terrain is no longer a visual object — it is a linear numerical dataset in memory.


5. Index-Based Vertex Mapping

Each vertex has a fixed index inside the array.

This allows conversion between:

  • 1D array index
  • 2D grid coordinates

Formulas:

  • Row = index / width
  • Column = index % width

This mapping enables:

  • Direct vertex access
  • Predictable terrain structure
  • Efficient procedural iteration

6. Grid Traversal System

Terrain generation uses nested loops:

  • Outer loop → rows
  • Inner loop → columns

For each vertex:

  • Index is calculated
  • Position is retrieved
  • Height is modified
  • Buffer is updated

This is the core mechanism behind all procedural terrain generation systems.


7. Procedural Height Generation

Terrain elevation is computed using procedural methods such as:

  • Noise functions
  • Heightmaps
  • Mathematical formulas

The result is stored in:

  • vertex.y

This generates natural terrain formations including:

  • Mountains
  • Valleys
  • Plateaus
  • Flat regions

All generated dynamically at runtime.


8. Neighbor Relationship System

Each vertex has implicit neighbors:

  • Left
  • Right
  • Up
  • Down
  • (Optional diagonals)

Although no explicit links exist, adjacency is derived from index math.

This enables:

  • Smoothing algorithms
  • Erosion systems
  • Brush-based deformation
  • Local averaging operations

9. Performance Scaling Model

Terrain resolution directly impacts performance:

  • High segment count → more vertices → higher GPU cost
  • Low segment count → fewer vertices → faster rendering

This introduces a critical design decision in terrain engines:

Balance between visual fidelity and computational efficiency

10. Terrain as a Data Field

The fundamental paradigm shift:

Terrain is not geometry — it is a computational data field.

Each vertex is:

  • Addressable
  • Modifiable
  • Recomputable
  • Dynamically rendered

This allows full procedural control over world generation systems.


11. Real-Time GPU Update System

When vertex data changes:

  • Buffer attributes are updated
  • GPU is notified via update flags
  • Scene is re-rendered instantly

This enables:

  • Live terrain sculpting
  • Real-time procedural generation
  • Continuous visual feedback loops

12. System Summary

By the end of this lesson, learners understand:

  • PlaneGeometry represents a structured vertex grid
  • Terrain is a programmable dataset, not a static model
  • Vertices exist in a linear memory buffer
  • Index mapping connects 1D memory to 2D grid logic
  • Procedural generation is driven by nested loops and noise functions
  • Neighbor relationships enable simulation systems
  • Performance scales with vertex density
  • Terrain operates as a live, real-time data field

Students gain the ability to:

  • Understand terrain as a data-driven system
  • Manipulate BufferGeometry vertex data
  • Map grid-based structures in 3D space
  • Build procedural terrain generation logic
  • Design scalable WebGL terrain architectures
  • Implement real-time simulation foundations
